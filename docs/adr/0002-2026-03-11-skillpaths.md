# [2026-03-11]

## Skills should delegate file path resolution to claude.md

### Context:

Agent-infra skills need to reference template files at known paths, but skills are symlinked into ~/.claude/skills/ and must work regardless of where the repo is cloned. This creates a portability problem for any path reference inside a SKILL.md file.

Key concerns driving the architecture:

- skill files are symlinked, so relative paths from $CLAUDE_SKILL_DIR don't resolve to the repo
- skills are versioned source files; install-time mutation dirties the working tree
- template paths must resolve correctly at the moment Claude invokes the Read tool

### Decision

Skills reference templates by name only, directing Claude to find the path in the Template use section of claude.md. The claude.md file is generated at install time from a source file containing the AGENT_INFRA_DIR placeholder, resolved via sed to an absolute path. Skills carry no paths and require no install-time modification.

### Consequences

**Good:**
- skills are path-free and install-agnostic
- install does not dirty the working tree
- all path resolution is centralised in one generated file

**Bad:**
- skills depend on claude.md being loaded in context
- template references in skills are indirect, not self-contained

**Alternatives considered:**
- $CLAUDE_SKILL_DIR relative paths: symlink breaks relative resolution
- install-time sed on SKILL.md files: mutates versioned source files
- hardcoded absolute paths in skills: not portable across users or clone locations
